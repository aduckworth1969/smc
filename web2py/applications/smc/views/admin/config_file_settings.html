Help

File server settings allow you to specify a location for user home directories. This script will create the folder, set permissions for this student, and set quota options if specified. For this to work, the drive letter and the home folder need to be set. WinRM is used to run the required commands and will need to be enabled on your file server.

File Server Import Enabled: Turn on/off the ability to setup home folder for users during import.
File Server Login User: Use a LOCAL NOT DOMAIN admin account that is on the file server. This account needs priviledges to create user folder and set permissions. The account HAS to be LOCAL or it will not work!!!
File Server Login Pass: The password for the local account described above. Password is encypted using AES encryption.
File Server Address: Please enter the IP of the File Server where user home folders will be created. NOTE: Using the IP is more reliable. Default options for DNS lookup does not allow this server to look back into your network to resolve names so the IP of your domain controller is needed.
File Server Quota Drive: If you use quotas for user folders, the import tool can set those during import. This drive letter is the physical drive letter on the file server, NOT the drive letter the user will see for their home folder.
